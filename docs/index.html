What It Does

Kindex maintains a graph of nodes (concepts, decisions, skills, projects, sessions) connected by weighted edges. It provides:

🔍

Hybrid Search

FTS5 full-text + graph traversal, merged via Reciprocal Rank Fusion. Question classification adjusts result depth automatically.

📊

Context Tiers

Five levels — full, abridged, summarized, executive, index — auto-selected by token budget for optimal AI injection.

⚠

Operational Nodes

Constraints, directives, checkpoints, and watches that surface at the right time. Pre-deploy checklists built into your graph.

🔄

Session Learning

Ingests Claude Code sessions, GitHub issues, git commits, and Linear tickets. LLM-powered extraction with budget controls.

🔒

Audience Tenancy

Private, team, org, and public scoping. PII stripping on export. Audience-aware boundary enforcement.

🔗

.kin Inheritance

Composable context chains across repos and teams. Depth-first resolution with parent directory auto-walk.

🧠

Bridge Discovery

Finds cross-domain connections you didn't manually encode. Suggests edges, detects analogies, tracks expertise.

⏰

Weight Decay

Nodes and edges naturally fade unless accessed. The graph stays fresh without manual curation.

Quick Start

# Initialize a knowledge store $ kin init # Add knowledge $ kin add "Stigmergy is coordination through environmental traces" Created node: stigmergy-is-coordination-th (concept) # Search with hybrid FTS5 + graph traversal $ kin search stigmergy 0.85 stigmergy-is-coordination-th concept Stigmergy is coordination... # Get context for Claude Code injection $ kin context --topic stigmergy --level full # Track operational rules $ kin add "Never break the API contract" --type constraint --trigger pre-deploy # Ingest from all sources $ kin ingest all # Install Claude Code hooks (one-time setup) $ kin setup-hooks $ kin setup-cron

Architecture

SQLite + FTS5 primary store and full-text search nodes: id, title, content, type, weight, audience, domains, extra edges: from_id, to_id, type, weight, provenance fts5: content synced via triggers Retrieval pipeline: FTS5 BM25 ──┐ Graph BFS ──┼── RRF merge ── tier formatter ── context block (vectors) ──┘ | full | abridged | summarized | executive | index Feedback loop: SessionStart ──> kin prime ──> context injected PreCompact ──> kin compact-hook ──> knowledge captured Cron (30min) ──> kin cron ──> decay, ingest, health checks

Claude Code Integration

One-time setup injects context at session start and captures knowledge before compaction:

# Install hooks into Claude Code settings $ kin setup-hooks Installed SessionStart hook: kin prime Installed PreCompact hook: kin compact-hook # Install periodic maintenance $ kin setup-cron Installed launchd plist (runs every 30 minutes)

Or configure manually in ~/.claude/settings.json:

{ "hooks": { "SessionStart": [{ "matcher": "", "hooks": [{ "type": "command", "command": "kin context --level executive", "timeout": 5000 }] }], "PreCompact": [{ "matcher": "", "hooks": [{ "type": "command", "command": "kin compact-hook --emit-context", "timeout": 10000 }] }] } }
